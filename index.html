HTML/CSS Cheatsheet

HTML

HTML Tags

Tag Name Description
<em> Emphasis Emphasizes text in browser, usually italicized by default.
<strong> Strong Browsers render this tag as bold.
<div> Div A container that divides into sections.
<br> Line Break Creates a break in text (addresses, poems), does not need a closing tag.
<img> Image The src contains the URL for the image, does not have a closing tag.
<span> Span An inline container for text, used for styling purposes.

Table Tags

Tag Name Description
<table> Table The wrapper element for all HTML tables.
<thead> Table Head The set of rows defining the column headers in a table.
<th> Table Head Data Table data container for the header.
<tbody> Table Body The set of rows containing actual tabel data.
<tr> Table Row The table row container.
<td> Table Data The table data container.
<tfoot> Table Footer The set of rows defining the footer in a table.

Table Attributes

Attribute Name Description
colspan Column Span Defines how many columns a td element should span.
rowspan Row Span Defines how many rows a td element should span.

Semantic HTML

Semantic HTML uses elements to describe what an element does or where it's placed. Some examples are: <header>, <nav>, <main>, and <footer>

Tag Description
<video> Adds video to a site, req. a closing tag.
<audio> Adds audio to a site, req. a closing tag.
<embed> Can be used to add any type of media, is a self closing tag.
<figure> Used to encapsulate media like images, diagrams, etc.
<figcaption> Used to describe media in the <figure>, used within the <figure> element.
<section> Defines elements in a doc. like chapters, or headings with the same theme.
<article> Used to hold content that makes sense on its own (articles, blogs, comments), can be used in/out of <section> element.
<aside> Used to mark additional info that can enhance another element, but is not req. to understand it (sidebar, ads).

HTML Forms

Tag Name Description
<textarea> Textarea Used to create a text-box for multi-line inputs (comment sections). It supports the rows and cols attributes that determine height/width, this can be adjusted in browser by user, but initial size is set by rows and cols.
<datalist> Datalist A basic search/autofill function by pairing a <input> and <datalist>, to pair them the <input>'s list must match the id of the <datatlist>. The datalist element stores a list of <option>s. The list is shown as a dropdown.
<select> Select Used to create a dropdown list, the choices for the dropdown is created by the <option> elements. There can only be one <option> selected at a time.
<form> Form Used to collect and send info to an external source, using the action attribute of the form (in the form of a URL, which also req. a method assigned a HTTP verb), and using the various input elements. Req. closing tag.
<input> Input Used to render a variety of input fields, including text fields, checkboxes, buttons, etc. inside the <form> element. The <input> element has a type that determines how it gets rendered. Self-closing tag.
Input attributes
name In order for a form to send data, it needs to be put in key-value pairs. This is done by setting the name attribute of the input element. The name will become the key and the user input will become the value assocsiated to it.(example: name="username") Must have an ID for form submission as well (id="username")
type="text" Creates a single row input field where users can type in. The value of the name and value attribute are sent as key-value pair when submitted.
type="number" Allows users to only enter numbers and special characters.
type="password" Creates a row for users to be able to input sensitve info in a censored format. The key-value pair is submitted with the actual value, not the censored value.
type="submit" Creates a submit button that, by default, will submit the <form> and execute the action.
type="checkbox" Using this attribute will render a single checkbox item, to create a group make sure they all use the same name attribute.
type="range" Creates a slider that acts as a selector between a minimum and a maximum value, rep. by the min and max attributes. The slider can be adjusted to move in different increments by using the step attribute.
type="radio" Creates a single radio button, multiple radio buttons of a related topic are give the same name value. Only a single option can be chosen from a group of radio buttons (i.e. pickup or delivery).
required Specifies that a <input> must include a value.
maxlength/minlength For input fields with type="text" that can specify the maximum/minimum number of characters that can be entered.

CSS

Syntax and Selectors

Name Description
Class/ID Classes can be resused, and use .example in CSS. ID are unique and should only be used for one element and use #example in CSS.
Type Selectors Type selectors are the same as HTML tags, but without angle brackets. Example: h1, p, table, etc.
Groups of CSS selectors Using a comma seperated list to group selectors to the same rules. Example: h1, h2.
Chaining Selectors Similar to groups w/o commas, the rule will only apply to elements that match all the criteria. Example: h3.example.
Selector Specificity Specificity is a ranking system for elements to determine which rule will apply. From most specific to least: ID selector, class selector, and type selector.

Visual Rules

CSS Declarations
Name Description
!important Rule Used to override anyother selectors regardless of specificity. Good practice to avoid using this rule.
font-family Used to set typeface of text. It must be browser available, incase of it not being available can set up backup typefaces. When using multi-word font names it must be wrapped in quotes.
font-size Used to set text sizes.
font-weight Used to set the boldness of text, uses keywords like normal or bold.
font-style Determines the font style in which text will appear, it accepts italic as a value.
line-height Declares the vertical spacing between lines of text. Accepts both unitless numbers as a ratio (2) and unit numbers (12px), but does not accept negative numbers.
text-align Sets alignment of text using keywords.
color Used to change color of text, can be named colors or use hex/rgb code.
background-color Changes background color of elements.
background-image Sets background image of element, uses a URL as the syntax. Ex: url("example.jpg").
opacity Controls transparency of an element. 0 being transparent to 1 opaque.

The Box Model

Name Description
box-sizing Default value is content-box, which renders the actual size of the element including the content box; but not the paddings and borders. border-box renders the actual size of an element including the content box, paddings, and borders.
overflow Determines how the browser will handle content too large for its container. By default, it is set to visible. Can also be set to hidden or scroll.
visibility Used to render hidden objects invisible to the user, without removing them from the page.
Margin Collapse Happens when the top and bottom margins of blocks are combined into a single margin equal to the largest margin. Only happens to vertical margins. auto keyword is used to horizontally center an element in its container.
Height and Width Maximums/Minimums min-width and min-height used to set minimums of an element's box. max-width and max-height used to set maximums of an element's box.

Display and Positioning

Name Description
z-index Specifies how far back or how far forward an element will appear on a web page when it overlaps other elements. Uses integer values, can be pos or neg, element with the highest z-index value is at the foreground, while the lowest will be at the back.
position Allows for positioning of HTML elements on a web page. Can be set to static, relative, absolute or fixed. fixed pins the element to a specific spot, regardless of scrolling (navigation bars).
position: absolute Allows an element to ignore sibling elements and be positioned to its closest parent element that is positioned with relative or absolute. By using top, left, bottom and right, the element can be positioned anywhere as expected.
position: relative Positions the element to where it would have originally been on the web page. Req. offset properties to determine the actual position of the element.
display Determines the type of render block for an element. Most common values are block, inline, and inline-block.
float Determines how far left or right an element should float wihtin its parent element. It can be used with the values left and right. Req. that the width of the container must be specified.
clear Used with float to specify how an element should behave with other elements in the same container. Determines on which sides floating elements are allowed to float.

Colors

Name Description
Hexadecimal Can represent sixteen different values using 0-9 and a-f. Composed of 6 characters, each group of two represents a value between 0-255 for red, green, and blue. When both characters of all three are repeated, can be abbrivated to only three values. Example: #0000ff = #00f.
RGB Declared by rgb() syntax. Has three values inside, each representing one of the three main colors, the values range from 0-255.
HSL Declared by hsl() syntax. Contains three values: hue (the color itself), saturation (intensity), and lightness. Hue values range from 0-360, while saturation and lightness are percentages.
